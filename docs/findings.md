## 6. 発見した問題と対策

### Julesの特性
- **@メンションでトリガーされない** — プロアクティブスキャンのみ
- **botコメントを無視する** — Codex CRの指摘は人間ユーザー（Claude/muumuu8181）経由で転記が必要
- **PRの90%にバグ混入** — Review→Fixサイクル必須
- **マージコンフリクトの自動解決が弱い** — 2/34 PRでコンフリクト未解決

### Codex CRの特性
- **Precision 100%** — 誤指摘なし（実測データ）
- **Recall 85%** — 15%のバグを見逃す可能性
- **レスポンスが速い** — `@codex review`後1〜5分で返答
- **CR枠はタスク枠と独立** — レビュー専業なら枯渇しない

### 自動マージの課題
- GitHubのauto-mergeにはブランチ保護ルールが必要
- `gh auth`に`workflow`スコープが無いと`.github/workflows/`をpushできない
- コンフリクト解決は`.coverage`ファイルが主な原因（Julesが各コンポーネントで生成）
